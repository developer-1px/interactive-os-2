# CMS Landing Sections — Design Spec

> Discussion: CMS 콘텐츠 섹션은 웹 페이지인데 앱 UI 토큰을 그대로 쓰고 있어서 구리다. 앱과 완전 분리된 독립 토큰 + 섹션별 카드/그리드 베리에이션 필요. 토큰 교체로 톤 전환 가능한 구조.

## 1. Purpose

- **풀려는 문제:** CMS 앱 안에서 편집되는 콘텐츠 섹션(랜딩 페이지)이 앱 UI와 같은 토큰을 써서 "관리 도구 안에 갇힌" 느낌. 웹 페이지다운 시각적 독립성이 없다.
- **대상 사용자:** CMS 콘텐츠를 보는 최종 방문자 (프레젠트 모드). CMS 관리자가 편집 중 프리뷰로 보는 경우도.
- **사용 맥락:** CMS 캔버스 내 렌더링 + 프레젠트 모드 풀스크린. 앱 크롬(사이드바/툴바/디테일패널)은 앱 토큰 유지, 캔버스 내부만 이 토큰 사용.

상태: 🟢

## 2. Tone

> 하나의 극단을 선택한다. "적당히"는 금지.

- **선택된 방향:** editorial-technical + Apple-style card composition. 아이콘 중심(사진 없음). 카드 사이즈/배치 믹스로 시각적 리듬. 넉넉한 여백, 선명한 타이포 위계, 절제된 장식.
- **레퍼런스:** Apple 제품 랜딩 (카드 사이즈 믹스, 여백, 아이콘 비주얼), Stripe 홈 (타이포 위계, 기술적 권위), Linear 홈 (다크 톤 + 카드 구성)
- **안티 레퍼런스 (이렇게는 안 됨):** SaaS 다크 랜딩의 보라 그라데이션 + 글로우 패턴. 사진 히어로. 일러스트 과다. 동일 사이즈 카드 반복.
- **앱 UI와의 관계:** 완전 별개 세계관. 앱 크롬은 industrial-utilitarian, 콘텐츠는 독립.
- **비주얼 어휘:** 아이콘(lucide 등) + 타이포그래피. 사진/일러스트 없음.

상태: 🟢

## 3. Constraints

- **프레임워크:** React + CSS Modules (기존 `PageVisualCms.module.css` 대체)
- **기존 디자인 시스템:** 앱 토큰(`tokens.css`)을 참조하지 않는다. 콘텐츠 전용 토큰 파일 신규 생성.
- **성능 제약:** 외부 폰트 추가 시 FOUT 주의. 가급적 시스템 폰트 또는 이미 로드된 Manrope 활용 검토.
- **접근성 요구:** WCAG 2.1 AA 대비 (4.5:1 text, 3:1 UI). prefers-reduced-motion 존중.
- **구조적 제약:** 토큰 값 교체만으로 톤 전환 가능해야 함 — CSS 셀렉터나 레이아웃을 변경하지 않고.

상태: 🟢

## 4. Differentiation

> 이 인터페이스를 본 사람이 딱 하나 기억하는 것

- **기억점:** 섹션마다 달라지는 카드 그리드 리듬. full-width → 2-col → 3-col → 1 large + 2 small 등 배치 변주 자체가 차별점. 별도 장식/기믹 없이 구성의 다양성으로 기억.

상태: 🟢

## 5. Typography

- **Display 글꼴:** Manrope (이미 로드됨, geometric sans로 Apple 스타일 궁합)
- **Body 글꼴:** Manrope
- **Mono 글꼴:** 앱과 동일 SF Mono 계열 (코드 스니펫 있을 경우만)
- **Size scale:** 앱보다 훨씬 큰 스케일. display: 48~64px, section-title: 32~40px, body: 16~18px, caption: 12~14px. 앱 최대(29px)보다 콘텐츠 최소 display가 더 큼.
- **Weight 체계:** display 700~800, title 700, body 400, caption 500. Weight 대비를 크게 잡아 위계 강조.

상태: 🟢

## 6. Color & Theme

- **지배색:** 뉴트럴 (흑/백/회색). 섹션 배경은 뉴트럴 밝기 차이로만 구분.
- **악센트:** 단일 색상 1개. 토큰 교체로 전체 톤 전환. 기본값은 별도 정의 (앱 indigo와 같을 필요 없음).
- **다크/라이트 정책:** 자체 다크/라이트. 앱 테마와 독립. 토큰 파일에서 dark/light 두 세트 정의.
- **CSS 변수 체계:** `--landing-*` 네임스페이스. 앱 토큰과 이름 겹침 없음.
- **의미색 (success/error/warning):** 불필요. 콘텐츠 랜딩에 상태 피드백 없음.

상태: 🟢

## 7. Motion

- **전략:** 범위 밖. 현재 스코프에서 모션은 다루지 않음.
- **고임팩트 모먼트 (페이지 로드, 전환):** N/A
- **Hover/Focus 반응:** N/A
- **라이브러리:** N/A

상태: ⬜ (범위 밖)

## 8. Spatial Composition

- **밀도:** spacious. 카드 내부 여백 넉넉, 섹션 간 간격 넓음.
- **레이아웃 접근:** Apple 스타일 카드 그리드 믹스. variant별 카드 사이즈/span 조합이 다름. full-width → 2-col → 3-col → 1 large + 2 small 등. 같은 그리드를 연속 반복하지 않음.
- **여백 전략:** 섹션 내부 padding 넉넉(64~80px 상하), 카드 간 gap 16~24px, 카드 내부 padding 24~32px. 콘텐츠 컬럼 max-width는 variant별로 다를 수 있음(720~1080px).
- **그리드 vs 자유 배치:** 그리드 기반. variant별 column/span 차이로 다양성.

상태: 🟢

## 9. Backgrounds & Visual Details

- **배경 처리:** 단색만. 카드 배경색 변주(밝은/약간 어두운/악센트 tint)로 깊이감. 그라데이션/글로우 없음.
- **텍스처/패턴:** 없음. 클린.
- **그림자 체계:** 카드에만 미묘한 그림자. 섹션 배경에는 그림자 없음.
- **장식 요소:** 아이콘만. 사진/일러스트/기하학적 장식 없음.

상태: 🟢
