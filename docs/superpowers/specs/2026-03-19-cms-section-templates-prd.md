# CMS Section Templates — PRD

> Discussion: 디자이너가 완성한 섹션을 템플릿으로 등록. 관리자는 "새 섹션 추가" 시 템플릿 종류를 선택. 기존 6개(Hero, Stats, Features, Workflow, Patterns, Footer)로 시작. 시스템이 먼저, 템플릿 수는 나중에 늘린다.

## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| M1 | 관리자가 새 섹션을 추가하려 함 | 사이드바 [+] 클릭 | 템플릿 선택 UI가 나타나고, 6가지 종류 중 선택 가능 |
| M2 | 관리자가 템플릿을 선택함 | 종류를 클릭하고 확인 | 해당 템플릿의 기본 구조(필드+자식 엔티티)가 store에 생성되고 캔버스에 표시 |
| M3 | 디자이너가 새 섹션 디자인을 만듦 | 템플릿을 등록하면 | [+] 선택 UI에 새 종류가 추가됨 |

상태: 🟢

## 2. 인터페이스

### 템플릿 선택 UI

- **트리거**: 사이드바 [+] 버튼
- **형태**: 사이드바 [+] 위치에서 위로 뜨는 드롭업 팝업 — 6개 종류를 프리뷰와 함께 나열
- **behavior**: `listbox` 또는 `grid` (종류가 많아지면 2열 그리드)
- **plugins**: 없음 — 읽기 전용 선택

| 입력 | 조건 | 결과 |
|------|------|------|
| ↑↓ (또는 ←→) | 템플릿 선택 UI 열린 상태 | 템플릿 간 포커스 이동 |
| Enter | 템플릿에 포커스 | 선택 확정, 섹션 생성, UI 닫힘 |
| Escape | 템플릿 선택 UI | UI 닫힘, [+] 버튼으로 포커스 복귀 |
| 클릭 | 템플릿 항목 | 선택 확정, 섹션 생성 |

상태: 🟢

## 3. 산출물

### 템플릿 정의

각 템플릿은 **섹션의 엔티티 구조를 정의하는 팩토리**:

| variant | 생성되는 구조 | 기본 자식 |
|---------|-------------|----------|
| `hero` | 섹션 + badge + title + subtitle + cta | 4개 필드 |
| `stats` | 섹션 + stat 카드 N개 | 4개 stat |
| `features` | 섹션 + header(label+title+desc) + card N개 (각 icon+title+desc) | header + 4개 카드 |
| `workflow` | 섹션 + header + step N개 (num+title+desc) | header + 4개 스텝 |
| `patterns` | 섹션 + header + pattern pill N개 | header + 14개 패턴 |
| `footer` | 섹션 + brand + links | brand + link 그룹 |

- **팩토리 함수**: variant를 받아 새 ID를 생성하고, 엔티티 + relationships를 반환
- **기본값**: 기본 locale(ko)에 placeholder 텍스트, 나머지 locale은 빈 문자열
- **추가 위치**: 사이드바에서 현재 선택된 섹션의 다음 위치에 삽입

### 템플릿 등록 구조

- 템플릿 정의는 코드 레벨 (이 단계에서 관리자 UI로 등록하지 않음)
- 디자이너가 만든 새 템플릿 = 개발자가 팩토리 + 렌더러를 추가

상태: 🟢

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| 같은 종류 섹션 여러 개 추가 | 허용. 각각 독립된 엔티티 (고유 ID) |
| 템플릿 선택 후 undo | 생성된 섹션 전체(엔티티+관계) 한 번에 롤백 |
| 삭제된 섹션과 같은 종류 재추가 | 새로운 엔티티로 생성 (삭제된 것과 무관) |

상태: 🟢

## 5. 금지

| # | 하면 안 되는 것 | 이유 |
|---|---------------|------|
| N1 | 관리자가 템플릿 구조를 수정 (필드 추가/삭제) | 디자이너가 정한 구조 보존 |
| N2 | 관리자가 런타임에 새 템플릿 종류 등록 | 이 단계에서 템플릿은 코드 레벨. 나중에 확장 가능 |
| N3 | 빈 섹션 (템플릿 없이) 추가 | 모든 섹션은 반드시 템플릿 기반 |
