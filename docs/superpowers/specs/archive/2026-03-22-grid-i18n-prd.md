# Grid 범용화 + CMS i18n Table 적용 — PRD

> Discussion: 구글 시트처럼 셀 단위 네비게이션·선택·편집이 되는 범용 Grid를 만들고, CMS i18n table에 적용. 모든 plugin 수용 가능해야 하며, 적용처마다 plugin 조합 선택. NormalizedData 단일 모델. CMS engine 공유.

## ① 동기

| # | Given | When | Then | 역PRD |
|---|-------|------|------|-------|
| M1 | Grid.tsx가 core(select+navigate)만 지원 | 개발자가 F2로 셀 편집, Delete로 행 삭제, Alt+↑↓로 리오더를 시도 | 아무 일도 안 일어남 — editing keyMap 없음 | |
| M2 | CMS i18n sheet가 plain HTML table | 관리자가 키보드로 셀 간 이동 시도 | 불가 — 탭으로만 이동 가능, 방향키 네비게이션 없음 | |
| M3 | TreeGrid.tsx에는 enableEditing + editingKeyMap 패턴 존재 | Grid에도 동일 패턴이 필요 | Grid가 TreeGrid와 동일한 수준의 plugin 지원을 해야 함 | |
| M4 | i18n sheet가 자체 useState로 editing/draft 관리 | 캔버스에서 undo하면 | i18n sheet의 편집 이력은 별도 — 통합 undo 불가 | |

완성도: 🟢

## ② 산출물

| 산출물 | 설명 | 역PRD |
|--------|------|-------|
| **Grid.tsx 확장** | enableEditing prop + editingKeyMap (F2, Delete, Alt+↑↓). TreeGrid과 동일 패턴 | |
| **ColumnDef 확장** | `{ key, header, field? }` — field가 있으면 rename 시 해당 entity field를 편집. 없으면 `cells[colIndex]` 직접 편집 | |
| **Grid renderCell 확장** | editing 상태(renaming)를 감지하여 Aria.Editable 렌더링 가능하도록 renderCell에 state 전달 | |
| **i18n 어댑터** | `translatableEntriesToGrid(store): NormalizedData` — CMS store entities → Grid용 NormalizedData 변환 | |
| **CmsI18nSheet 교체** | plain HTML table → Grid 컴포넌트 + CMS engine 공유 | |
