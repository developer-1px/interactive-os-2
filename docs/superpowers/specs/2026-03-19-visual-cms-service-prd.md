## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| M1 | CMS 관리자가 랜딩페이지를 편집하려 함 | 앱을 열면 | Figma Slides 스타일 CMS 편집 화면이 메인. 좌상단 햄버거(≡)로 쇼케이스 접근 |
| M2 | 관리자가 콘텐츠(텍스트/이미지/순서)만 변경하고 싶음 | 섹션을 선택하면 | CRUD + 서식 도구만 제공되고, 디자인(레이아웃/색상/패딩) 변경 수단은 없다 |
| M3 | 디자이너가 새 섹션 디자인을 완성함 | 템플릿으로 등록하면 | 관리자가 "새 섹션 추가"에서 해당 템플릿을 선택할 수 있다 |
| M4 | 관리자가 다국어 콘텐츠를 관리함 | 언어 탭을 전환하면 | 같은 레이아웃에 해당 언어 텍스트가 표시되고, i18n 시트와 양방향 동기화된다 |
| M5 | 관리자가 다양한 디바이스에서의 모습을 확인하려 함 | 뷰포트를 전환하면 | 모바일/태블릿/데스크톱 프레임으로 프리뷰된다 |
| M6 | 관리자가 게시 후 최종 모습을 확인하려 함 | [▶] 프레젠트를 누르면 | 모든 크롬(사이드바, 툴바, 포커스 링)이 사라지고 최종 결과물만 보인다 |

## 2. 인터페이스

> CMS 전용 레이아웃의 인터페이스. 개별 기능의 상세 인터페이스는 각 PRD 참조.

| 입력 | 조건 | 결과 |
|------|------|------|
| 앱 진입 (`/`) | 기본 상태 | CMS 편집 화면이 메인. ActivityBar 없음 (햄버거 메뉴로 접근) |
| 좌상단 [≡] 클릭 | CMS 편집 중 | ActivityBar가 오버레이/드로어로 열림 (쇼케이스 레이어 목록) |
| 오버레이 외부 클릭 또는 Escape | ActivityBar 드로어 열린 상태 | 드로어 닫히고 CMS 복귀 |
| 드로어에서 다른 레이어 클릭 | 드로어 열린 상태 | 해당 쇼케이스 페이지로 이동 (기존 데모 앱 레이아웃) |
| 쇼케이스 ActivityBar에서 CMS 클릭 | 쇼케이스 보는 중 | CMS 레이아웃으로 복귀, 이전 편집 상태 유지 |
| [▶] 클릭 | CMS 편집 중 | 프레젠트 모드 진입 — 모든 크롬 숨김, 최종 결과물만 풀스크린 표시 |
| Escape | 프레젠트 모드 | 프레젠트 종료, CMS 편집 화면 복귀 |

### CMS 레이아웃 구조 (Figma Slides 스타일)

```
┌────────────────────────────────────────────────────┐
│ [≡]  [ko ▾]                    [📱 💻 🖥️]  [▶]  │  ← 얇은 상단 툴바
├───────┬────────────────────────────────────────────┤
│  ┌─┐  │                                           │
│  │▮│  │                                           │
│  └─┘  │                                           │
│  ┌─┐  │            캔버스                          │
│  │▮│  │       (spatial nav CMS)                    │
│  └─┘  │                                           │
│  ┌─┐  │         ┌────────────────┐                │
│  │▮│  │         │  플로팅 툴바    │                │
│  └─┘  │         └────────────────┘                │
│       │                                           │
│  [+]  │                                           │
└───────┴────────────────────────────────────────────┘
```

- **[≡]**: 햄버거 → ActivityBar 쇼케이스 드로어
- **[ko ▾]**: 언어 드롭다운 (→ i18n PRD)
- **[📱 💻 🖥️]**: 뷰포트 전환 (→ viewport PRD)
- **[▶]**: 프리뷰/프레젠트 모드
- **섹션 사이드바**: 썸네일 스트립 + [+] 추가 버튼 (→ sidebar PRD)
- **플로팅 툴바**: 캔버스 하단 가운데 떠있음, 컨텍스트 반응 (→ toolbar PRD)

상태: 🟢

## 3. 산출물

> CMS 전용 레이아웃 산출물. 개별 기능의 산출물은 각 PRD 참조.

- CMS 전용 라우트 (`/`) 및 레이아웃 컴포넌트
- 얇은 상단 툴바 (햄버거 + 언어 드롭다운 + 뷰포트 토글 + 프레젠트)
- 섹션 사이드바 (썸네일 스트립)
- 캔버스 영역 (spatial nav CMS 확장)
- 플로팅 툴바 (캔버스 하단 센터, 떠있음)
- 햄버거 드로어 (ActivityBar 쇼케이스 목록, 오버레이)

상태: 🟢

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| 햄버거 드로어 열린 상태에서 캔버스 클릭 | 드로어 닫힘, 캔버스 포커스 복귀 |
| 햄버거 드로어 열린 상태에서 Escape | 드로어 닫힘, 햄버거 버튼으로 포커스 복귀 |
| 뷰포트 프리뷰(모바일) 중 편집 | 축소된 캔버스 안에서 spatial nav 동작 |
| 브라우저 폭이 좁을 때 | 섹션 사이드바 접힘, 캔버스 전체 사용 |
| 쇼케이스에서 CMS 복귀 | 이전 편집 상태(포커스 위치, 스크롤, 선택) 유지 |
| 플로팅 툴바가 캔버스 콘텐츠와 겹칠 때 | 툴바가 콘텐츠 위에 떠있음 (캔버스 레이아웃에 영향 없음) |

상태: 🟢

## 5. 금지

| # | 하면 안 되는 것 | 이유 |
|---|---------------|------|
| N1 | 디자인 변경 UI 제공 (색상, 폰트, 패딩, 레이아웃 조정) | 관리자에게 디자인 변경 가능성 = 공포. 템플릿이 디자인을 고정 |
| N2 | 섹션 내부 구조 변경 (필드 추가/삭제/재배치) | 디자이너가 정한 구조를 관리자가 깨뜨릴 수 없음 |
| N3 | ActivityBar를 CMS에 직접 노출 | CMS는 캔버스 중심. ActivityBar는 햄버거 드로어 뒤에 숨김 |
