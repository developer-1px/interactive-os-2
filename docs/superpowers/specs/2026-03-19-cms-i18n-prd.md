# CMS i18n — PRD

> Discussion: 콘텐츠 다국어. 상단 언어 드롭다운으로 전환 + 스프레드시트형 번역 시트. 에디터↔시트 양방향 동기화. key-value 기반.

## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| M1 | 관리자가 한국어 콘텐츠를 편집 중 | 상단 `[ko ▾]`에서 `en` 선택 | 같은 레이아웃에 영어 텍스트 표시. 빈 필드는 빈 상태로 표시 |
| M2 | 관리자가 영어 캔버스에서 텍스트를 편집함 | rename으로 값 수정 | i18n 시트의 해당 키 `en` 컬럼에 자동 반영 |
| M3 | 관리자가 번역 전체 현황을 파악하려 함 | i18n 시트를 열면 | 모든 키가 행, 언어가 열인 스프레드시트. 빈 셀 = 미번역 |
| M4 | 관리자가 시트에서 직접 번역을 입력함 | 시트 셀 편집 | 캔버스에서 해당 언어로 전환하면 입력한 값이 표시 |

상태: 🟢

## 2. 인터페이스

### 언어 드롭다운 (상단 툴바)

- **behavior**: `combobox` 또는 단순 `<select>` — 언어 수가 적으므로 (3~5개)
- 선택 시 캔버스의 모든 텍스트가 해당 locale의 값으로 교체

### i18n 시트

- **behavior**: `grid` (row = 키, col = 언어)
- **plugins**: `core()` — 셀 간 포커스 이동. rename plugin으로 셀 편집

```
┌──────────────┬─────────────────┬─────────────────┬──────────────────┐
│ Key          │ ko              │ en              │ ja               │
├──────────────┼─────────────────┼─────────────────┼──────────────────┤
│ hero-title   │ 헤드리스 ARIA…  │ Headless ARIA…  │                  │
│ hero-subtitle│ 모든 ARIA role… │ Build fully…    │                  │
│ stat-patterns│ 14              │ 14              │ 14               │
│ ...          │ ...             │ ...             │ ...              │
└──────────────┴─────────────────┴─────────────────┴──────────────────┘
```

### i18n 시트 접근 방식

| 입력 | 조건 | 결과 |
|------|------|------|
| 상단 툴바 시트 토글 버튼 클릭 | CMS 편집 중 | 캔버스 하단에 i18n 시트 패널이 슬라이드 업으로 열림 |
| ↑↓←→ | 시트에 포커스 | grid behavior — 셀 간 이동 |
| Enter / F2 | 시트 셀에 포커스 | rename으로 셀 편집 진입 |
| Escape | 시트에 포커스 | 시트 닫기 또는 캔버스 복귀 |

### 데이터 모델

- store의 entity data에 locale별 값을 저장
- 예: `{ type: 'text', value: { ko: '제목', en: 'Title', ja: '' } }`
- 기존 `value: string`에서 `value: Record<Locale, string>`으로 확장
- 현재 locale은 UI 상태 (store 밖)

상태: 🟢

## 3. 산출물

- **언어 드롭다운** — 상단 툴바 좌측, 현재 locale 표시 + 전환
- **i18n 시트 패널** — grid behavior로 전체 번역 매핑 표시
- **데이터 확장** — entity data의 텍스트 값을 locale map으로 변환
- **양방향 동기화** — 캔버스 편집 → 시트 반영, 시트 편집 → 캔버스 반영 (같은 store)
- **미번역 표시** — 빈 문자열인 셀을 시각적으로 구분 (시트에서 하이라이트)

상태: 🟢

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| 미번역 필드를 캔버스에서 볼 때 | 기본 locale(ko) 텍스트를 흐리게(opacity) 표시 — 구조 파악 + 미번역 인지 |
| 새 섹션 추가 시 | 기본 locale(ko)에만 템플릿 기본값, 나머지 locale은 빈 문자열 |
| 섹션 삭제 시 | 모든 locale의 해당 키가 함께 삭제 |
| 언어 추가/삭제 | 이 PRD 범위 밖 — 지원할 언어는 설정으로 고정 |
| 숫자/아이콘 등 비텍스트 필드 | locale 무관 — 모든 언어에서 동일 값 |

상태: 🟢

## 5. 금지

| # | 하면 안 되는 것 | 이유 |
|---|---------------|------|
| N1 | locale별 다른 레이아웃/디자인 | 레이아웃은 하나. 텍스트만 바뀜 |
| N2 | 자동 번역 기능 | CMS는 콘텐츠 관리. 번역은 외부 도구 |
| N3 | i18n 시트 전용 store | 하나의 앱 = 하나의 store |
| N4 | 현재 locale을 store에 저장 | 보기 모드일 뿐, 콘텐츠가 아님 |

## 6. 검증

| # | 시나리오 | 예상 결과 |
