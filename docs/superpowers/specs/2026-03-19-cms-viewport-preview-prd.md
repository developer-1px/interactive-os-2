# CMS Viewport Preview — PRD

> Discussion: 상단 툴바의 뷰포트 토글로 모바일/태블릿/데스크톱 프리뷰. 캔버스를 CSS transform으로 축소하여 디바이스 프레임 안에 표시.

## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| M1 | 관리자가 모바일에서의 모습을 확인하려 함 | 상단 툴바에서 📱 클릭 | 캔버스가 모바일 폭으로 축소되고 디바이스 프레임으로 감싸짐 |
| M2 | 관리자가 데스크톱 전체 편집으로 돌아가려 함 | 🖥️ 클릭 | 캔버스가 원래 크기로 복귀 |
| M3 | 관리자가 축소된 뷰포트에서도 편집하려 함 | 모바일 프리뷰 상태에서 클릭/키보드 | spatial nav가 축소된 캔버스 안에서 정상 동작 |

상태: 🟢

## 2. 인터페이스

### 뷰포트 옵션

| 아이콘 | 이름 | 폭 |
|--------|------|-----|
| 📱 | Mobile | 375px |
| 💻 | Tablet | 768px |
| 🖥️ | Desktop | 100% (기본값) |

### 인터랙션

| 입력 | 조건 | 결과 |
|------|------|------|
| 뷰포트 아이콘 클릭 | 상단 툴바 | 캔버스가 해당 폭으로 전환. CSS transform scale로 축소 |
| 캔버스 클릭/키보드 | 뷰포트 축소 상태 | spatial nav 정상 동작 (scale이 이벤트에 영향 없음) |
| 프레젠트 모드 진입 | 뷰포트 축소 상태 | 프레젠트는 항상 실제 브라우저 크기로 표시. 뷰포트 설정 무시 |

상태: 🟢

## 3. 산출물

- **뷰포트 래퍼 컴포넌트** — 캔버스를 감싸는 컨테이너
  - 선택된 뷰포트에 따라 `max-width` + `CSS transform scale` 적용
  - 센터 정렬 (좌우 여백 자동)
- **뷰포트 토글** — 상단 툴바 우측의 3개 아이콘 버튼 그룹
- **상태**: 뷰포트 선택은 로컬 UI 상태 (store 밖 — 콘텐츠가 아닌 보기 모드)

상태: 🟢

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| 모바일 뷰에서 가로로 넓은 섹션 (stats 4열) | CSS 반응형에 따라 리플로우 (실제 모바일과 동일) |
| scale 적용 시 클릭 좌표 | pointer-events가 scale된 요소 안에서 정상 동작 (브라우저 기본) |
| 플로팅 툴바 위치 | 뷰포트 프레임 밖, 원래 위치 고정 |
| 섹션 사이드바 썸네일 | 뷰포트 변경과 무관 — 항상 전체 프리뷰 |
| 브라우저 폭이 뷰포트보다 좁을 때 | 추가 축소 또는 가로 스크롤 |

상태: 🟢
