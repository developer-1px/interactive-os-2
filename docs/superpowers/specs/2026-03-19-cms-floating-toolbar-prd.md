# CMS Floating Toolbar — PRD

> Discussion: Figma 하단 플로팅 바 모델. 캔버스 하단 센터에 떠있고, spatial nav 깊이/선택에 따라 CRUD 도구가 바뀐다. Rich text 기각 — plain text(rename) 편집만.

## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| M1 | 관리자가 섹션을 선택함 (Root 깊이) | 포커스가 섹션에 있으면 | 툴바에 섹션 CRUD 도구가 표시된다 (복제, 삭제, ↑↓ 리오더) |
| M2 | 관리자가 카드/아이템을 선택함 (Collection 깊이) | 포커스가 컬렉션 아이템에 있으면 | 툴바에 아이템 CRUD 도구가 표시된다 (추가, 복제, 삭제, ↑↓ 리오더) |
| M3 | 관리자가 필드에 진입함 (Fields 깊이) | 포커스가 필드에 있으면 | 툴바에 필드 수준 도구가 표시된다 (rename 편집은 F2/Enter/더블클릭으로 진입) |
| M4 | 아무것도 선택되지 않음 | 포커스가 없거나 캔버스 밖 | 툴바는 보이되 비활성/빈 상태 |

상태: 🟢

## 2. 인터페이스

> toolbar behavior + 깊이별 컨텍스트 전환

### behavior + plugin 조합

- **behavior**: `toolbar` (role: toolbar, childRole: button)
- **plugins**: 없음 — 툴바 자체는 읽기 전용. 버튼 클릭이 캔버스 store에 command를 dispatch

### 컨텍스트별 도구

| 깊이 | 포커스 대상 | 툴바 내용 |
|------|-----------|----------|
| Root | 섹션 (hero, features...) | `[복제] [삭제] [↑ 위로] [↓ 아래로]` |
| Collection | 카드, 스텝, 패턴 등 | `[+ 추가] [복제] [삭제] [↑ 위로] [↓ 아래로]` |
| Fields | 텍스트/이미지 필드 | 비활성 (필드 구조는 템플릿 고정, 값 편집은 rename으로 직접 진입) |
| Edit | 인라인 편집 중 (rename) | 비활성 |
| 없음 | 포커스 없음 | 툴바 보이되 모든 버튼 비활성 |

### 인터랙션

| 입력 | 조건 | 결과 |
|------|------|------|
| ← / → | 툴바에 포커스 | toolbar behavior 기본 — 이전/다음 버튼 |
| Enter / Space | 툴바 버튼에 포커스 | 해당 command를 캔버스 store에 dispatch |
| Escape | 툴바에 포커스 | 캔버스로 포커스 복귀 |
| Tab | 캔버스에서 | 툴바로 포커스 이동 (자연스러운 탭 순서) |
| 클릭 | 툴바 버튼 | command dispatch, 포커스는 캔버스에 유지 (클릭 후 캔버스 포커스 복원) |

상태: 🟢

## 3. 산출물

- **플로팅 툴바 컴포넌트** — `position: fixed`, 캔버스 하단 센터
  - `<Aria>` + toolbar behavior
  - 깊이 컨텍스트에 따라 버튼 세트 교체
- **컨텍스트 감지** — 캔버스의 현재 포커스 깊이(`__spatial_parent__` 기반)를 읽어 툴바 내용 결정
- **command dispatch** — 툴바 버튼이 캔버스의 store에 직접 command dispatch (같은 store)
- **시각**: 반투명 배경 + 그림자, 캔버스 콘텐츠 위에 떠있음 (z-index)

상태: 🟢

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| 캔버스 콘텐츠가 툴바와 겹칠 때 | 툴바가 위에 떠있음. 캔버스 레이아웃에 영향 없음 |
| 깊이 전환 시 (Enter/Escape) | 툴바 내용이 즉시 교체됨 (애니메이션 없이 또는 subtle fade) |
| 인라인 편집(rename) 진입 시 | 툴바 비활성. 편집 중 키 이벤트가 툴바에 누수되지 않음 |
| 삭제 버튼 클릭 후 | focusRecovery 미들웨어가 다음 포커스 결정 |
| 뷰포트가 모바일 프리뷰일 때 | 툴바는 프리뷰 프레임 밖, 원래 위치에 고정 |
| 프레젠트 모드 | 툴바 숨김 |

상태: 🟢

## 5. 금지

| # | 하면 안 되는 것 | 이유 |
|---|---------------|------|
| N1 | 디자인 변경 도구 (색상, 폰트, 패딩, 레이아웃) | 콘텐츠 관리 도구. 디자인은 범위 밖 |
| N5 | Rich text 서식 도구 (B/I/U/링크) | 기각됨. plain text 편집(rename plugin)만 지원 |
| N2 | 툴바 전용 store | 하나의 앱 = 하나의 store |
| N3 | 툴바가 캔버스 레이아웃을 밀어냄 | floating이므로 캔버스에 영향 없음 |
| N4 | 깊이와 무관한 고정 도구 세트 | 컨텍스트 반응이 핵심 — 불필요한 도구 노출은 혼란 |
