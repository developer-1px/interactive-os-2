# Axis v2: 5축 모델 — PRD

> Discussion: metadata 행동 플래그를 축 옵션으로 흡수하여, 축 조합만으로 pattern을 완전 정의할 수 있게 한다. 11축 → 5축(navigate, select, activate, expand, trap).

## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 현재 Axis = keyMap only이고 행동 플래그(followFocus, selectionMode 등)가 metadata에 분산 | 새 pattern을 정의하려 할 때 | 축 조합 + metadata 양쪽을 알아야 해서 "축의 조합 = pattern" 공식이 성립하지 않는다 |
| 2 | activateFollowFocus 축이 activate와 코드 100% 동일 | followFocus 동작을 이해하려 할 때 | metadata에 숨어있어서 축 이름만으로 동작을 파악할 수 없다 |
| 3 | 11개 축이 존재하지만 navV/navH, selectToggle/selectExtended 등 같은 카테고리의 변형이 별개 파일 | 축 시스템을 학습하려 할 때 | 동일 관심사가 흩어져서 옵션 차이를 파악하기 어렵다 |
| 4 | 쇼케이스에서 축만 보여주면 pattern의 절반(metadata 행동)이 안 보임 | 축 쇼케이스를 만들 때 | "왜 축으로 분해했는가"를 전달할 수 없다 |

상태: 🟢

## 2. 인터페이스

> 이 PRD의 인터페이스는 **축 factory의 API 계약** — 어떤 옵션을 넣으면 어떤 행동이 나오는가.

### navigate(options?)

| 입력 (옵션) | 조건 | 결과 (keyMap + config) |
|---|---|---|
| `orientation: 'vertical'` | 기본값 | ↑↓ Home End 바인딩. focusStrategy.orientation = 'vertical' |
| `orientation: 'horizontal'` | | ←→ Home End 바인딩. focusStrategy.orientation = 'horizontal' |
| `orientation: 'both'` | | ↑↓←→ 모두 prev/next 바인딩 |
| `wrap: true` | | focusNext/focusPrev에 `{ wrap: true }` 전달 |
| `grid: { columns: N }` | orientation은 자동으로 'vertical' | ↑↓←→ Home End Mod+Home Mod+End 바인딩. config.colCount = N |

### select(options?)

| 입력 (옵션) | 조건 | 결과 |
|---|---|---|
| `mode: 'multiple'` | 기본값 | Space → toggleSelect. config.selectionMode = 'multiple' |
| `mode: 'single'` | | Space → select (교체). config.selectionMode = 'single' |
| `extended: true` | mode: 'multiple'일 때만 유효 | Shift+↑↓ Shift+Home/End 범위 선택 추가 |

### activate(options?)

| 입력 (옵션) | 조건 | 결과 |
|---|---|---|
| (옵션 없음) | 기본값 | Enter Space → activate() |
| `onClick: true` | | config.activateOnClick = true |
| `followFocus: true` | | config.followFocus = true (포커스 이동 시 자동 활성화) |
| `toggleExpand: true` | | activate()가 expand/collapse를 토글. config.expandable = true |

### expand(options?)

| 입력 (옵션) | 조건 | 결과 |
|---|---|---|
| `mode: 'arrow'` | 기본값 | → = expand/focusChild, ← = collapse/focusParent. config.expandable = true |
| `mode: 'enter-esc'` | | Enter = 자식 진입(spatialCommands), Esc = 부모 복귀. config.expandable = true |

### trap(options?)

| 입력 (옵션) | 조건 | 결과 |
|---|---|---|
| `escape: true` | 기본값 | Esc → collapse/close |
| `tabCycle: true` | | Tab/Shift+Tab이 컨테이너 내부에서 순환 (?) |
| `lightDismiss: true` | | 외부 클릭 시 닫기 (?) |

> `tabCycle`과 `lightDismiss`는 현재 미구현. → 별도 PRD 범위. 이 사이클에서는 `trap({ escape: true })`만.

### focusStrategy 기본값 규칙

- navigate 축이 **있으면** → `roving-tabindex` (orientation은 navigate 옵션에서)
- navigate 축이 **없으면** → `natural-tab-order` (dialog, disclosure 등)

## 3. 산출물

### Axis 타입 확장

```
// v1
type Axis = Record<string, (ctx) => Command | void>

// v2
interface Axis {
  keyMap: Record<string, (ctx) => Command | void>
  config?: Partial<PatternConfig>
}

interface PatternConfig {
  focusStrategy: FocusStrategy
  expandable: boolean
  selectionMode: SelectionMode
  activateOnClick: boolean
  followFocus: boolean
  colCount: number
}
```

### PatternMetadata 축소 → Identity

```
// v1 PatternMetadata (8개 필드)
{ role, childRole, focusStrategy, expandable, selectionMode, activateOnClick, followFocus, colCount, ariaAttributes }

// v2 Identity (3개 필드)
{ role, childRole, ariaAttributes }
```

### 파일 구조 변경

```
src/interactive-os/axes/
├── compose-pattern.ts    — Axis 타입 확장, Identity 타입, config 머지 로직
├── navigate.ts           — navigate() factory (navV, navH, navVhUniform, navGrid 통합)
├── select.ts             — select() factory (selectToggle, selectExtended 통합)
├── activate.ts           — activate() factory (activate, activateFollowFocus 통합)
├── expand.ts             — expand() factory (depthArrow, depthEnterEsc 통합)
└── trap.ts               — trap() factory (focusTrap 확장)

삭제 대상:
├── nav-v.ts              → navigate({ orientation: 'vertical' })
├── nav-h.ts              → navigate({ orientation: 'horizontal' })
├── nav-vh-uniform.ts     → navigate({ orientation: 'both', wrap: true })
├── nav-grid.ts           → navigate({ grid: { columns: N } })
├── depth-arrow.ts        → expand({ mode: 'arrow' })
├── depth-enter-esc.ts    → expand({ mode: 'enter-esc' })
├── select-toggle.ts      → select()
├── select-extended.ts    → select({ extended: true })
├── activate-follow-focus.ts → activate({ followFocus: true })  **삭제**
└── focus-trap.ts         → trap()
```

### 17개 behavior 재작성

모든 behavior가 5축 factory를 사용하도록 변경. 예:

```
// listbox.ts v1
composePattern({ role, childRole, focusStrategy, selectionMode: 'multiple', activateOnClick: true, ariaAttributes },
  selectExtended, selectToggle, activate, navV)

// listbox.ts v2
composePattern({ role: 'listbox', childRole: 'option', ariaAttributes },
  navigate({ orientation: 'vertical' }),
  select({ mode: 'multiple', extended: true }),
  activate({ onClick: true }),
)
```

### composePattern 변경

1. metadata → identity (role, childRole, ariaAttributes만)
2. 각 축에서 keyMap을 수집 + config를 머지
3. 머지된 config를 AriaBehavior에 전달

### useAria / Aria 컴포넌트

변경 없음 — AriaBehavior 인터페이스의 출력은 동일. composePattern이 축 config를 AriaBehavior 필드로 변환하므로 소비자 측 변경 불필요.

상태: 🟢

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| navigate() 옵션 없이 호출 | 기본값 orientation: 'vertical', wrap: false |
| select() 옵션 없이 호출 | 기본값 mode: 'multiple', extended: false |
| activate() 옵션 없이 호출 | Enter/Space만, onClick: false, followFocus: false, toggleExpand: false |
| expand() 옵션 없이 호출 | 기본값 mode: 'arrow' |
| trap() 옵션 없이 호출 | 기본값 escape: true, tabCycle: false, lightDismiss: false |
| select({ extended: true, mode: 'single' }) | extended 무시 — single에서 범위 선택은 의미 없음 |
| 같은 카테고리 축 2개 (예: navigate 2번) | composePattern이 config를 뒤의 것으로 덮어씀. keyMap은 chain of responsibility |
| config 충돌 (축 A가 expandable: true, 축 B가 expandable: false) | 마지막 축의 config 승리 (순서 의존) |
| v1 축 코드를 직접 import하는 외부 코드 | 삭제 시 빌드 에러. v1 축 파일 삭제 전 전수 grep 필요 |
| navigate 축 없는 pattern | focusStrategy = natural-tab-order 자동 적용 |

상태: 🟢

## 5. 금지

| # | 하면 안 되는 것 | 이유 |
|---|---------------|------|
| 1 | v1 축 파일을 남겨두기 (deprecate만) | 두 시스템 공존은 혼란. 마이그레이션 완료 후 즉시 삭제 |
| 2 | AriaBehavior 인터페이스 변경 | useAria/Aria 컴포넌트는 AriaBehavior를 소비. 출력 인터페이스를 바꾸면 영향 범위가 폭발 |
| 3 | 축 factory 안에서 DOM/React 의존 | 축은 순수 함수. DOM은 useAria/Aria의 영역 |
| 4 | tabCycle, lightDismiss를 이 사이클에서 구현 | 별도 PRD 범위. trap({ escape: true })만 이 사이클 |
| 5 | barrel export | 프로젝트 규칙 |
