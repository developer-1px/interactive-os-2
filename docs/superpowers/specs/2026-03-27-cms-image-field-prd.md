# CMS Image Field — PRD

> Story S3: 관리자는 섹션의 이미지를 교체할 수 있다. 현재 CMS는 아이콘(Lucide 키)만 지원. 실제 이미지(사진/배너) 교체 불가.

## ① 동기

### WHY

- **Impact**: 랜딩페이지에 사진/배너가 필수인데 CMS에서 이미지 교체가 불가. 관리자가 아이콘만 바꿀 수 있어 콘텐츠 편집기로서 불완전.
- **Forces**: 기존 아이콘 선택 UI(IconField)는 잘 동작. `fieldType` 메타 시스템도 있어서 새 필드 타입 추가가 자연스러움. 단, 서버 업로드는 API 연동이므로 지금은 범위 밖.
- **Decision**: `image` fieldType 추가 + ImageField 컴포넌트 + Data URL 로컬 저장. 기각: (a) 아이콘을 이미지로 교체 → 기존 아이콘 시스템 파괴, (b) 외부 URL 입력만 → 파일 선택 UX 없이 불편.
- **Non-Goals**: 서버/CDN 업로드, 이미지 리사이즈/크롭, 이미지 라이브러리(갤러리) 관리.

### 시나리오

| # | Given | When | Then | 역PRD |
|---|-------|------|------|-------|
| 1 | Hero 섹션에 배너 이미지 필드가 있음 | 상세 패널에서 이미지 필드 클릭 → 파일 선택 | 선택한 이미지가 프리뷰로 표시되고 캔버스에 즉시 반영됨 | |
| 2 | 카드에 이미 이미지가 설정되어 있음 | 다른 이미지로 교체 | 기존 이미지가 새 이미지로 바뀌고 캔버스에 즉시 반영 | |
| 3 | 이미지가 설정된 카드 | 이미지 제거 | 이미지가 빈 상태로 돌아감 (플레이스홀더 표시) | |

완성도: 🟡

## ② 산출물

| 산출물 | 설명 | 역PRD |
|--------|------|-------|
| `image` fieldType | cms-schema.ts의 FIELD_TYPES에 'image' 추가 | |
| ImageField 컴포넌트 | CmsDetailPanel 내 이미지 선택/프리뷰/제거 위젯 | |
| 이미지 렌더링 | cms-renderers.tsx에서 image fieldType 필드를 `<img>` 로 렌더링 | |
| 이미지 사용 노드 | 기존 노드 중 이미지가 적합한 곳에 image 필드 추가 (hero 배너, article 썸네일 등) | |

완성도: 🟡

## ④ 경계

| 극단 조건 | 현재 상태 | 왜 이렇게 동작해야 하는가 | 예상 동작 | 결과 상태 | 역PRD |
|----------|----------|------------------------|----------|----------|-------|
| 매우 큰 파일 선택 (10MB+) | 파일 선택 | Data URL은 메모리 부담. 프로덕션에서는 서버 업로드로 전환할 것 | 경고 없이 수용 (Data URL). 성능 문제는 서버 업로드 전환 시 해결 | 이미지 저장됨 | |
| 이미지가 아닌 파일 선택 | 파일 선택 다이얼로그 | accept="image/*"로 필터링되지만 사용자가 우회 가능 | accept 속성으로 1차 필터. 비이미지는 무시 | 변경 없음 | |
| 이미지 필드가 다국어(locale)인 경우 | 다국어 편집 | 이미지는 보통 언어 무관 (같은 사진). 단, alt 텍스트는 다국어 가능 | 이미지 src는 비다국어, alt는 다국어 필드로 분리 | — | |
| 실행취소 후 이미지 복원 | 이미지 교체 후 Cmd+Z | 히스토리에 Data URL이 저장됨 | 이전 이미지로 복원됨 | 이전 이미지 표시 | |
