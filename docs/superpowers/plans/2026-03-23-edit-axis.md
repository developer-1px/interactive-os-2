# Edit Axis Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create an `edit()` axis that owns editing keyMap bindings (F2, Enter, Delete, Alt+Arrow), eliminating the hardcoded `editingKeyMap` constants from Grid.tsx, ListBox.tsx, TreeGrid.tsx. `replaceEditPlugin` is a standalone plugin export.

**Architecture:** The edit axis returns a standard `StructuredAxis` (keyMap only) — no new type needed. The replace-mode printable-key plugin is exported separately as `replaceEditPlugin()`. UI components pass `edit: true` to behavior factories and add `replaceEditPlugin()` to their plugins array when editing is enabled. `listbox`/`treegrid` change from `const` to function (16+1 callers updated mechanically). Grid cell clipboard (Mod+C/V with colIndex) stays as component-level keyMap override.

**Tech Stack:** TypeScript, React, Vitest, @testing-library/react

## Key Design Decisions

### 1. Edit axis keyMap
```
F2           → renameCommands.startRename(focused)
Delete       → crudCommands.remove(focused)
Alt+Up       → dndCommands.moveUp(focused)
Alt+Down     → dndCommands.moveDown(focused)
```

### 2. Enter key: always included
Current behavior in ALL three UI components: Enter → startRename (overrides activate in ListBox/TreeGrid via keyMap precedence). The edit axis always binds Enter to maintain this.

### 3. Tree-specific extensions
```
Alt+Left  → dndCommands.moveOut(focused)   (tree: true)
Alt+Right → dndCommands.moveIn(focused)    (tree: true)
```

### 4. Replace-mode plugin is separate
`replaceEditPlugin()` is a standalone plugin export, NOT embedded in the axis return type. UI components add it to their plugins array. This avoids extending `StructuredAxis` type or leaking plugin through behavior return type.

### 5. Grid cell clipboard stays as component override
Mod+C/V depends on `ctx.grid?.colIndex` — grid-specific, not in edit axis.

### 6. listbox/treegrid callers (17 files)
Mechanical change: `listbox` → `listbox()`. All callers that don't need editing pass no options.

```typescript
// src/interactive-os/axes/edit.ts
import type { StructuredAxis } from './composePattern'
import type { Plugin } from '../core/types'
import { crudCommands } from '../plugins/crud'
import { renameCommands, RENAME_ID } from '../plugins/rename'
import { dndCommands } from '../plugins/dnd'
import { isPrintableKey } from '../plugins/typeahead'
import type { CommandEngine } from '../core/createCommandEngine'
import { FOCUS_ID } from '../plugins/core'

interface EditOptions {
  /** Add Alt+Left/Right for tree indent/outdent */
  tree?: boolean
}

export function edit(options?: EditOptions): StructuredAxis {
  const keyMap = {
    'F2': (ctx) => renameCommands.startRename(ctx.focused),
    'Enter': (ctx) => renameCommands.startRename(ctx.focused),
    'Delete': (ctx) => crudCommands.remove(ctx.focused),
    'Alt+ArrowUp': (ctx) => dndCommands.moveUp(ctx.focused),
    'Alt+ArrowDown': (ctx) => dndCommands.moveDown(ctx.focused),
  } satisfies StructuredAxis['keyMap']

  if (options?.tree) {
    Object.assign(keyMap, {
      'Alt+ArrowLeft': (ctx) => dndCommands.moveOut(ctx.focused),
      'Alt+ArrowRight': (ctx) => dndCommands.moveIn(ctx.focused),
    } satisfies StructuredAxis['keyMap'])
  }

  return { keyMap }
}

/** Printable-key → replace-mode rename plugin. Add to plugins array when editing is enabled. */
export function replaceEditPlugin(): Plugin {
  return {
    onUnhandledKey(event: KeyboardEvent, engine: CommandEngine): boolean {
      if (!isPrintableKey(event)) return false
      const store = engine.getStore()
      if (store.entities[RENAME_ID]?.active) return false
      const focusedId = (store.entities[FOCUS_ID]?.focusedId as string) ?? ''
      if (!focusedId) return false
      engine.dispatch(renameCommands.startRename(focusedId, { replace: true, initialChar: event.key }))
      return true
    },
  }
}
```

```typescript
// grid.ts — add edit axis import, add to options and axes array
import { edit as editAxis } from '../axes/edit'

export function grid(options: { columns: number; tabCycle?: boolean; edit?: boolean }): AriaBehavior {
  const axes = [
    select(),
    navigate({ grid: { columns: options.columns, tabCycle: options.tabCycle } }),
  ]
  if (options.edit) {
    axes.push(editAxis())
  }
  return composePattern({ role: 'grid', childRole: 'row', ariaAttributes: ... }, ...axes)
}
```

- [ ] **Step 2: Update listbox — const to function**

```typescript
// listbox.ts — change export const to export function
import { edit as editAxis } from '../axes/edit'

export function listbox(options?: { edit?: boolean }): AriaBehavior {
  const axes = [
    select({ mode: 'multiple', extended: true }),
    activate({ onClick: true }),
    navigate({ orientation: 'vertical' }),

  if (options?.edit) {
    axes.push(editAxis())

  return composePattern({ role: 'listbox', ... }, ...axes)
}
```

Enter key conflict: `composePattern` tries handlers in array order — first non-undefined return wins. Edit axis must be **first** in the axes array so its Enter handler runs before activate's Enter. This matches current behavior where component editingKeyMap overrode behavior keyMap.

```typescript
const axes = []
if (options?.edit) axes.push(editAxis())  // FIRST — edit Enter wins over activate Enter
axes.push(select(...), activate(...), navigate(...))
```

2. **useTabList.ts**: NOT changed. Its editingKeyMap has different semantics (Alt+Left/Right = horizontal reorder via moveUp/moveDown, not indent/outdent). Separate concern.

3. **Grid useMemo deps**: `enableEditing` must be in behavior useMemo dependency array. Plan shows this explicitly.
