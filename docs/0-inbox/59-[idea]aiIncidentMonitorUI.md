# AI 장애 모니터링 UI 구상 — 2026-03-26

## 배경

팀장이 원하는 "대단한 것" = AI를 이용한 장애 모니터링/대응 도구. 기존 데모는 Playwright 주기 감시 + 이미지 캡쳐 분석의 1page 웹사이트. "시각화 + 채팅 + 모니터링"을 엮어서 MCP/CLI로는 불가능한 가치를 만드는 것이 목표.

시장 조사(docs/3-resources/32-[tooling]aiIncidentManagementUI.md) 결과, Chat과 시각화가 진정으로 융합된 인터페이스는 시장에 빈자리로 확인됨.

## 내용

### 핵심 차별화: "동시에 보고, 대화하고, 조작하는 화면"

기존 도구의 한계:
- Chat-First (incident.io, Rootly): 동시 정보 비교 불가
- Dashboard-Embedded (Datadog, Grafana): AI가 생성만, 조작은 분리
- Agent 자율 (PagerDuty): 블랙박스, 실시간 추적 불가

### 화면 구성

```
┌─────────────────────────────────────────────────────────┐
│  [모니터링 바]  서비스A ● 서비스B ● 서비스C ◉ 서비스D ●  │  ← 상시 갱신
├──────────────────────────────────┬──────────────────────┤
│                                  │                      │
│  [타임라인]                       │  [캡쳐 비교]          │
│  ──●──────●──────◉──────●───▶   │  Before │ After      │
│    09:00  09:15  09:23  09:30   │  [img]  │ [img]      │
│                  ↑ 장애 시작     │                      │
│    ← → 키보드로 시점 이동         │  AI: "이 시점에서     │
│                                  │  header 영역이       │
│  [이벤트 목록]                    │  사라졌습니다"        │
│  09:23 배포 v2.3.1              │                      │
│  09:23 에러율 급증               │                      │
│  09:20 config 변경              │                      │
│                                  │                      │
├──────────────────────────────────┴──────────────────────┤
│  [AI 채팅]                                              │
│  > 이 시점부터 뭐가 바뀌었어?                              │
│  AI: 09:23에 v2.3.1 배포됨. header 컴포넌트 변경.        │
│      에러율이 0.1% → 12.3%로 급증. [타임라인에서 보기]     │
│  > 롤백하면 될까?                                        │
│  AI: v2.3.0으로 롤백 권장. 영향 범위: header, nav.        │
└─────────────────────────────────────────────────────────┘
```

### 4개 영역의 역할

| 영역 | 역할 | 동기화 |
|------|------|--------|
| 모니터링 바 | 서비스 전체 상태 한눈에 | ↑↓ 키로 서비스 전환 → 아래 영역 모두 변경 |
| 타임라인 | 시간축 탐색 | ←→ 키로 시점 이동 → 캡쳐 비교 동기화 |
| 캡쳐 비교 | before/after 시각 증거 | 타임라인 시점에 연동 |
| AI 채팅 | 자연어 질의 + 조치 제안 | AI 응답의 시점 링크 → 타임라인 점프 |

### 키보드 인터랙션

| 키 | 효과 |
|----|------|
| `←` `→` | 타임라인 시점 이동, 캡쳐 동기화 |
| `↑` `↓` | 모니터링 바 서비스 전환 |
| `/` | AI 채팅 포커스 |
| `Enter` (AI 응답 링크) | 타임라인 해당 시점 점프 |
| `Space` | 캡쳐 전체화면 |

### interactive-os 대응

| 기능 | interactive-os |
|------|---------------|
| 타임라인 탐색 | TreeGrid + spatial navigation |
| 서비스 전환 | multi-zone + cross-boundary |
| AI 채팅 | Command + Plugin |
| 캡쳐 비교 | multi-view + Entity store |
| 실시간 갱신 | Entity store + 선언적 뷰 |

### 미해결 갭

1. AI 응답 → 타임라인 하이라이트 연결 메커니즘
2. 캡쳐 비교 UI (슬라이더 vs 나란히 vs 오버레이)
3. Playwright 캡쳐 데이터 구조 확인 필요
4. AI 백엔드 (이미지 분석 모델, 응답 시간)
