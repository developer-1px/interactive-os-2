# Visual CMS — User Story Map

## Needs (욕구)

| ID | 페르소나 | 욕구 | 설명 |
|----|---------|------|------|
| N1 | 관리자 | 콘텐츠를 편집하기 원한다 | 개발자 없이 텍스트/이미지 교체 |
| N2 | 관리자 | 페이지 구조를 변경하기 원한다 | 개발자 없이 섹션 추가/삭제/순서변경 |
| N3 | 관리자 | 변경 결과를 확인하고 배포하기 원한다 | 개발자 없이 프리뷰 + 라이브 반영 |
| N4 | 관리자 | 다국어 콘텐츠를 관리하기 원한다 | 개발자 없이 언어별 콘텐츠 전환 |
| N5 | 디자이너 | 섹션 템플릿을 등록하기 원한다 | 만든 디자인을 관리자가 쓸 수 있게 등록 |
| N6 | 디자이너 | 등록한 템플릿이 그대로 유지되기 원한다 | 관리자가 디자인을 깨뜨리지 못하는 보장 |

## Stories (스토리 = 릴리즈 단위)

| ID | 욕구 | 스토리 | 수용조건 | PRD | 상태 |
|----|------|--------|---------|-----|------|
| S1 | N1 | 관리자는 편집할 페이지를 선택할 수 있다 | 페이지 목록이 보이고, 선택하면 섹션 구성이 표시된다 | — | ⬜ |
| S2 | N1 | 관리자는 섹션의 텍스트를 교체할 수 있다 | 텍스트 필드를 선택하고 내용 변경. plain text만, 서식 불가 | — | ⬜ |
| S3 | N1 | 관리자는 섹션의 이미지를 교체할 수 있다 | 이미지 필드를 선택하고 다른 이미지로 교체 | — | ⬜ |
| S4 | N2 | 관리자는 템플릿에서 새 섹션을 골라 추가할 수 있다 | 템플릿 목록에서 선택, 원하는 위치에 삽입 | — | ⬜ |
| S5 | N2 | 관리자는 섹션을 삭제할 수 있다 | 선택한 섹션 제거, 실수 방지 확인 | — | ⬜ |
| S6 | N2 | 관리자는 섹션 순서를 변경할 수 있다 | 드래그 또는 키보드로 섹션 위치 이동 | — | ⬜ |
| S7 | N3 | 관리자는 변경 결과를 프리뷰로 확인할 수 있다 | 데스크톱/모바일 뷰포트 전환 가능 | — | ⬜ |
| S8 | N3 | 관리자는 변경사항을 라이브에 배포할 수 있다 | 변경 전/후 diff 확인 후 원클릭 배포 | — | ⬜ |
| S9 | N4 | 관리자는 언어를 전환하여 해당 언어의 콘텐츠를 편집할 수 있다 | 언어 드롭다운 선택 → 해당 언어 콘텐츠 표시 및 편집 | — | ⬜ |
| S10 | N5 | 디자이너는 완성된 섹션 디자인을 템플릿으로 등록할 수 있다 | 등록된 템플릿이 관리자의 템플릿 목록에 나타남 | — | ⬜ |
| S11 | N6 | 관리자가 편집할 때 디자인(레이아웃/색상/폰트)은 변경되지 않는다 | 편집 가능 영역이 콘텐츠 필드로만 제한됨 | — | ⬜ |
| S12 | N1 | 관리자는 페이지의 메타 정보(SEO, URL, OG)를 편집할 수 있다 | 메타 필드를 수정하면 프리뷰/배포에 반영 | — | ⬜ |
| S13 | N2 | 관리자는 섹션 내 아이템(카드, 항목 등)을 추가/삭제/순서변경할 수 있다 | 컬렉션 아이템 CRUD. 슬롯(고정 구조)은 변경 불가 | — | ⬜ |
| S14 | N1 | 관리자는 편집 작업을 되돌리거나 다시 실행할 수 있다 | 실행취소/다시실행으로 실수 복구 | — | ⬜ |
| S15 | N1 | 관리자는 노드를 복사하여 붙여넣을 수 있다 | 같은 타입끼리만 붙여넣기 가능 | — | ⬜ |
