# Core

> 데이터 구조와 엔진. Entity 기반 정규화 store + Command 기반 상태 변환.

## 주기율표

| 모듈 | 함수/타입 | 역할 | 상태 |
|------|----------|------|------|
| types | `Entity`, `NormalizedData`, `Command`, `BatchCommand`, `ROOT_ID` | 핵심 타입 정의 | 🟢 |
| createStore | `createStore()` | 정규화 데이터 store 생성, CRUD 연산 | 🟢 |
| createCommandEngine | `createCommandEngine()` | Command 파이프라인 실행, 플러그인 체인 | 🟢 |

## 의존 방향

```
types (Entity, Command, NormalizedData)
  ↓
createStore (데이터 조작)
  ↓
createCommandEngine (Command 실행 + Plugin 체인)
  ↓
plugins, hooks (소비자)
```

## 핵심 개념

- **Entity**: `{ id, data }` — 모든 데이터의 단위
- **NormalizedData**: `{ entities, relationships }` — 트리 구조의 정규화 저장
- **Command**: `(store) => store` — 순수 함수로 상태 변환
- **Plugin**: Command 실행 전후에 개입하는 미들웨어
