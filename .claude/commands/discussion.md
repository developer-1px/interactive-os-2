# Discussion — 의도 정렬 프로토콜

사용자의 요청을 바로 실행하지 않는다. 표면적 요청 너머의 숨겨진 의도를 파악하고, 충분히 정렬된 후에만 실행한다.

## 시작

1. **축적된 지식 로드** — 메모리 파일들과 `docs/0-inbox/`의 관련 지식 문서를 읽는다
2. **능동 탐색** — 코드 읽기, 웹 검색, 스킬 등 모든 도구를 사용해 맥락을 스스로 파악한다
3. 사용자에게 묻기 전에 스스로 채울 수 있는 gap은 먼저 채운다

## 루프

매 턴마다 반드시 아래 두 가지를 출력한다:

### 1. 이해도 테이블

```
| 필드 | 이해도 | 내용 |
|------|--------|------|
| **추측된 의도** | [■■■□□] 60% | 표면적 요청 너머의 근본적 목적 |
| **원칙/우선순위** | [■□□□□] 20% | 충돌 시 판단 기준, 사용자가 중요하게 여기는 것 |
| **맥락** | [■■□□□] 40% | 반문/교정에서 드러난 숨겨진 배경 |
| 목적 | [■■■□□] 60% | 명시적으로 말한 목표 |
| 요구사항 | [■□□□□] 20% | 구체적 결과물 |
| 제약 | [■■□□□] 40% | 하면 안 되는 것 |
| 성공 기준 | [□□□□□] 0% | 어떻게 되면 "됐다"인지 |
| 미확인 | — | 아직 모르는 것 목록 |
```

### 2. gap을 채우기 위한 질문

- 한 번에 하나의 질문만
- 가능하면 선택지(A/B/C)를 제공
- **현재 맥락을 근거로 자기 의견(추천)을 함께 제시한다** — 선택지만 나열하지 않음
- 스스로 채울 수 없는 gap만 질문한다

## 반문 감지

사용자가 반문하거나 "아니야"라고 할 때:
- 내 추측이 틀린 이유를 탐색한다
- 드러난 숨겨진 맥락을 **즉시** 기록한다 (메모리 또는 지식 문서)
- 이해도 테이블을 업데이트한다

## 종료 조건

- gap이 없다고 판단되면 → "go 해주세요" 제안
- 사용자가 **"go"**라고 말하면 → discussion 종료, 실행 전환

## "go" 후 실행

1. 정렬된 의도를 바탕으로 **적합한 스킬들을 자율 판단**하여 순서대로 실행한다
   - 예: brainstorming → writing-plans → subagent-driven-development
   - 예: 단순 작업이면 바로 구현
   - 스킬 선택과 순서는 LLM이 의도에 맞게 판단
2. ralph loop을 기동하여 여러 턴에 걸쳐 자율 완주한다
3. 변경사항이 없으면 ralph loop을 자동 종료한다

## 종료 시 학습

discussion이 끝나면 반드시:

1. **발견된 맥락** → 메모리에 저장 (feedback 또는 user 타입)
2. **발견된 원칙/우선순위** → 메모리에 저장
   - 기존 원칙과 충돌하면 사용자에게 확인 후 업데이트
3. **다음 discussion**에서 저장된 지식을 로드하고 시작

## 절대 하지 않는 것

- discussion 중 코드를 수정하지 않는다
- "~하겠습니다", "~로 하죠" 같은 결론을 내리지 않는다
- 사용자가 "go" 하기 전에 실행하지 않는다
- 이해도 테이블 없이 질문하지 않는다
