# Kanban Drag & Drop — PRD

> Discussion: 칸반 보드에서 카드를 마우스 드래그앤드롭으로 다른 컬럼으로 이동할 수 있어야 한다. 현재는 키보드(Alt+Arrow)로만 이동 가능.

## 1. 동기

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 칸반 보드에 카드가 있는 상태 | 사용자가 카드를 마우스로 드래그하여 다른 컬럼에 드롭 | 카드의 소속 컬럼이 변경되고, 드롭 위치에 삽입된다 |
| 2 | 칸반 보드에 카드가 있는 상태 | 사용자가 카드를 같은 컬럼 내에서 드래그하여 다른 위치에 드롭 | 카드의 순서(order)가 재계산된다 |
| 3 | 키보드만 사용하는 사용자 | Alt+Arrow로 카드를 이동 | 기존 동작이 그대로 유지된다 (회귀 없음) |
| 4 | 마우스 사용자가 드래그 중 | Escape를 누르거나 원래 위치에 드롭 | 드래그가 취소되고 원래 상태로 복귀 |

상태: 🟡

## 2. 인터페이스

|------|------|------|

## 3. 산출물

> 구조, 관계, 이름 — 파일/컴포넌트/데이터 스키마

**구조:** 칸반 보드는 가로로 컬럼(group)이 배치되고, 각 컬럼 안에 카드가 세로로 나열된다. 드래그앤드롭은 이 2차원 구조에서 카드를 컬럼 간/컬럼 내 이동시킨다.

**데이터 변경:** 드롭 시 기존 `dndCommands.moveTo(cardId, targetParentId, index)` 커맨드를 재사용한다. 새 커맨드를 만들지 않는다.

**UI 레이어:**
- Kanban.tsx에 드래그 이벤트 핸들러 추가 (onDragStart, onDragOver, onDrop, onDragEnd)
- 드래그 중 시각적 피드백: 드래그 중인 카드의 ghost 표시, 드롭 위치 인디케이터
- kanban.css에 드래그 관련 스타일 추가 (`.kanban-card--dragging`, `.kanban-column--drop-target`, `.kanban-drop-indicator`)

**데이터 모델 변경 없음:** 기존 NormalizedData의 relationships로 부모-자식 관계가 관리되며, 드래그앤드롭도 동일한 `moveTo` 커맨드를 dispatch한다.

## 4. 경계

| 조건 | 예상 동작 |
|------|----------|
| 빈 컬럼으로 드롭 | 카드가 해당 컬럼의 첫 번째 아이템으로 삽입 |
| 컬럼 헤더 위에 드롭 | 해당 컬럼의 마지막 위치에 삽입 (헤더는 드롭 대상이 아니라 컬럼 전체의 프록시) |
| 보드 바깥으로 드래그 후 mouseup | 드래그 취소 (원위치 복귀) |
| 카드가 rename 모드일 때 드래그 시도 | 드래그 시작하지 않음 (renaming이 우선) |
| 컬럼이 1개뿐일 때 | 컬럼 내 재정렬만 가능 |
| 매우 긴 컬럼에서 드래그 | 스크롤 영역 내에서 정상 동작, 자동 스크롤은 scope 외 (?) |
| 선택된 카드 여러 개를 드래그 | 선택된 모든 카드가 대상 컬럼으로 이동 (순서 유지) (?) |

## 5. 금지

| # | 하면 안 되는 것 | 이유 |
|---|---------------|------|
| 1 | 외부 라이브러리(react-dnd, dnd-kit 등) 도입 | interactive-os는 자체 ARIA 엔진 위에 구축. 외부 DnD 라이브러리는 자체 포커스/상태 관리를 가지고 있어 충돌한다 |
| 2 | 새 Command 타입 생성 | 기존 `dndCommands.moveTo`로 충분. 커맨드 중복은 undo/redo 일관성을 깨뜨린다 |
| 3 | 키보드 DnD 동작(Alt+Arrow) 변경 | 기존 키보드 이동과 마우스 드래그는 같은 커맨드를 사용하되, 트리거만 다르다. 키보드 동작 회귀 금지 |
| 4 | 드래그 중 포커스 이동 | 드래그 중에는 포커스가 원래 카드에 유지. 드래그 hover로 포커스가 따라가면 ARIA 상태가 꼬인다 |
| 5 | 컬럼 자체의 드래그 이동 | 이 PRD의 범위는 카드 이동만. 컬럼 순서 변경은 별도 PRD |
| 6 | behavior(kanban.ts)에 마우스 이벤트 로직 추가 | behavior는 keyMap 기반. 마우스 드래그 로직은 UI 컴포넌트(Kanban.tsx) 레이어에서 처리 |

상태: 🟡
