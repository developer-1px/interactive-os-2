## 5. 금지

## 구현 범위

이 PRD는 retro 스킬(`/Users/user/Desktop/aria/.claude/skills/retro/SKILL.md`)의 **Step 6: 경험 추출** 내 "빈도 기반 승격" 섹션을 확장하는 것이다. 구체적으로:

### 수정 대상 파일

| 파일 | 변경 내용 |
|------|----------|
| `.claude/skills/retro/SKILL.md` | Step 6 "빈도 기반 승격" 섹션에 (a) PRD 체크리스트 자동 추가 절차 (b) 구조화된 승격 이력 기록 포맷 추가 |

### 승격 시 PRD 체크리스트 추가 절차 (Step 6에 삽입)

빈도 3회 이상 승격 시 기존 3단계 뒤에 추가:

4. **PRD 체크리스트 연동**: 교훈의 도메인이 PRD 작성과 관련되는지 판정한다.
   - 관련 도메인: `prd/*`, `behavior/*`, `engine/*`, `store/*`, `plugin/*`, `ui/*` 등 설계 결정이 PRD에 반영되는 영역
   - 비관련 도메인: `test/*`, `css/*`, `build/*` 등 PRD 체크리스트로 검증할 수 없는 영역
   - 관련 시: 교훈을 `- [ ] {검증 질문}` 형태로 변환하여 PRD SKILL.md의 해당 항목 섹션에 추가
   - 추가 전: 기존 체크리스트와 의미적 중복 확인 → 중복이면 건너뜀
5. **승격 이력 기록**: experience_db.md의 해당 행에 구조화된 승격 이력 추가
   - 포맷: `→ feedback 승격됨 (YYYY-MM-DD, feedback: {파일 경로}, PRD: {추가 항목 또는 "해당 없음" 또는 "중복"})`

### 경험 DB 테이블 확장

현재 테이블 열: `# | 도메인 | 상황 | 갭 | 교훈 | 빈도 | 최근`

승격 이력은 기존 행 끝에 텍스트로 추가 (열 추가 아님 — 테이블 폭 유지):
```
| 11 | spatial/findNearest | ... | ... | ... | 3 | 2026-03-25 | → feedback 승격됨 (2026-03-25, feedback: memory/feedback_find_nearest_external_origin.md, PRD: "외부 원점 탐색 시나리오를 경계에서 다뤘는가?" 추가) |
```

### Knowledge 기반 설계 결정

- **K1**: 경험 DB → feedback 승격은 "삽질이 규칙이 될 만큼 반복"된 신호. 따라서 승격 자체는 자동이되, 이력을 남겨 승격의 맥락(언제, 왜, 어떤 체크리스트가 생겼는지)을 추적 가능하게 한다.
- **K2**: L2 수정은 PRD 스킬 체크리스트 추가. 승격된 교훈이 PRD 관련 도메인이면 자동으로 L2 수정(체크리스트 추가)까지 연결하여, 같은 갭이 PRD 작성 단계에서 사전 차단되도록 한다.
