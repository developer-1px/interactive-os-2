## 응답

코드를 보면 상황이 "DnD가 키보드로 안 된다"가 아니라 **"칸반에서는 되는데 다른 곳에서는 안 된다"**에 가깝다.

현재 상태를 정리하면:

1. **dnd 플러그인** (`plugins/dnd.ts`) — `moveUp`, `moveDown`, `moveOut`, `moveIn`, `moveTo` 5개 커맨드가 이미 있다. undo도 된다.
2. **kanban behavior** (`behaviors/kanban.ts`) — `Alt+Arrow` 4방향으로 키보드 DnD가 완전히 동작 중이다. 크로스 컬럼 이동(`Alt+Left/Right`)은 index 보존(가역적 동선)까지 구현되어 있고, 다중 선택 이동도 된다.
3. **treegrid, listbox 등** — dnd 플러그인을 등록해도 키바인딩이 연결되지 않은 상태.

그래서 질문의 핵심이 갈린다:

- **A) treegrid/listbox 같은 기존 behavior에도 키보드 DnD를 붙이고 싶다** — 각 behavior에 Alt+Arrow 축을 추가하는 문제. 기존 dnd 커맨드를 재사용하면 되니까 비교적 직선적이다.
- **B) 마우스 드래그 경험(드래그 프리뷰, 드롭 타겟 하이라이트 등)을 키보드로도 제공하고 싶다** — 이건 완전히 다른 문제. "grab 모드" 진입 → 방향키로 이동 → Enter로 드롭 같은 상태 머신이 필요하다.
- **C) 다른 생각이 있으시면**

**제 판단: A.** 이유: 현재 dnd 플러그인 설계가 이미 "커맨드 실행 = 즉시 이동"이고, kanban에서 이 패턴이 잘 동작하고 있다. 이 모델을 다른 behavior로 확산하는 게 자연스럽다. B는 시각적 피드백 레이어가 필요해서 복잡도가 크게 뛴다.
